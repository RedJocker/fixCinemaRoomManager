Description

Our venue is getting really popular. Customers now want to purchase tickets right from the app. Let's add a dialog box that will allow them to do it. Don't forget to display the current state of the reserved seating.

The best approach is to use DialogFragment that allows to easily avoid memory leaks and other issues with properly state handling due to device rotations and other android stuff.

Anyway, under the hood, it uses a default android.app.AlertDialog, so let's use it to avoid compatibility issues

Objectives

A click on the seat should display a dialog that contains a ticket price and two options: buy a ticket or cancel the purchase. If the ticket is already reserved, no dialog option should be available. For a better user experience, let's color seats in the cinema room. Make available seats green and booked seats gray.

1) The AlertDialog title should have a row and a seat number. An example template: Buy a ticket 1 row 6 seat;

2) The AlertDialog message should contain a ticket price. An example template: Your ticket price is 24.11$;

3) The AlertDialog should have a positive button with the Buy a ticket text;

4) The AlertDialog should have a negative button with the Cancel purchase text;

5) The booked seat should be disabled. The seat indicator R.id.cinema_room_place_indicator should be of the CardView type with a gray color (Color.DKGRAY or android.R.color.darker_gray) background. Use the setCardBackgroundColor method to set up a proper property.
